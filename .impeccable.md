# VeilFlow — Design

## Design Context

### Users
Three audiences, all institutional / professional, all in daytime focused work:
- **Issuers** — treasury / ops people setting up confidential distributions (investor airdrops, team vesting, payroll/disperse). They need confidence and precision; mistakes move real money.
- **Recipients** — investors / employees claiming an allocation only they can see.
- **Auditors / regulators** — granted read-only disclosure of a single figure; reading a compliance report, not operating.

Context of use: a serious financial tool, opened on a laptop during the workday, often alongside a wallet and a block explorer. Not a consumer app, not a degen dApp.

### Brand Personality
**Discreet · exacting · editorial.** VeilFlow should feel like a *confidential term sheet from a private bank* — composed, printed with care, quietly authoritative. It is a privacy product (amounts are encrypted end-to-end; the backend never sees plaintext) AND an institutional finance instrument (TokenOps GTM is Nomura/BlackRock). The interface earns trust by looking like a precise document, not a flashy app. Emotional goal: **calm confidence.** Never breathless, never crypto-loud.

### Aesthetic Direction
- **Theme:** Light. Warm ivory "paper", ink-black type. Institutional daytime reading; an honest, open UI for a product whose secrecy lives in the cryptography, not the chrome.
- **Flavor:** Editorial / private-bank. Hairline rules and generous whitespace over drop-shadowed cards. Letterspaced small-caps labels, folio/section numbering, serif display headlines, a narrow measure for prose.
- **Color:** Zama-aligned. Golden Zama **yellow** is the single accent, used like a **highlighter / wax seal** — rare and sharp (~10% weight). Ink-black + ivory paper carry everything else; neutrals are tinted warm toward the brand hue.
- **Type:** Display = **Libre Caslon Display** (the prospectus/legal-document voice). Serif body/lede = **Libre Caslon Text**. UI / data / body = **Schibsted Grotesk** (editorial-publishing grotesque). Ciphertext / addresses / hashes = **Fragment Mono** (typed-document machine voice). Tabular figures for amounts.
- **Anti-references:** generic shadcn SaaS (Inter + grayscale + shadowed rounded cards), neon-on-dark crypto, gradient text, glassmorphism, purple→blue gradients, identical icon-card grids, centered hero-metric templates.

### Signature
**Redaction bars.** Every encrypted value renders as a classified-document censor bar (solid ink). It **lifts (clip-path wipe) to reveal the figure only when the viewer decrypts it** — the FHE `userDecrypt` action mapped directly to un-redacting a confidential document. It is the core data primitive (every amount/balance/allocation), not decoration. Selection highlight is the Zama yellow highlighter. Status reads as a small engraved **seal/stamp**, not a pill.

### Design Principles
1. **Document, not dashboard.** Compose with rules, whitespace, and folio structure. Reach for a card only when something must literally be boxed. Never nest cards.
2. **Sealed by default; reveal is an act.** Confidential figures start redacted. Revealing is deliberate, animated, and scoped to the one who holds the key.
3. **Gold is the action.** Zama gold fills the primary/emphasis button (the one main action per region) and marks live/active states, the active nav, and key highlights. Secondary actions are outline/ghost/link; chrome stays ink on paper. Gold earns attention by being the thing you're meant to do — not decoration sprayed everywhere.
4. **Set numbers like a banker.** Tabular figures, mono for ciphertext/addresses, ink for amounts. Alignment and precision over ornament.
5. **Calm motion.** One orchestrated reveal beats scattered micro-interactions. Transform/opacity/clip-path only; exponential ease-out; honor reduced-motion.
